HTML & CSS: The journalist and the designer

Journalists are good at informing the public through dissemination of facts, in the process giving meaning to people's lives by clarifying what the consequences of those facts are.

Designers amplify this meaning by interpreting the story and presenting it in a manner that improves the uptake of the facts and helps generate emotions in the reader.

HTML is like the journalist, in that it supplies a browser with details of what specific pieces of content on the page are meant for. CSS is like the designer in that it helps create visual hierarchy and structures the layout of that content in aid of consuming and understanding the content more readily.

DOM: Trees, wonderful trees

The DOM represents the hierarchical structure of a document type using a tree diagram to present the location and order of specific elements/components of said document type. We can then use this tree diagram to directly interact with/manipulate these elements/components/objects through the use of the browser's DevTools interface.

For instance, we can isolate a button in our HTML, give it a class, and change its styling dependent on that class in the css, all from within the browser's DevTools thanks to the DOM.

Data access: Simpler than the Dewey Decimal system

An array is like 1 column bookCase[], one object per 'cubicle'. In order to access data in the array, you have to specify the position (using integers from 0 upwards) of the cubicle you want to access, which returns the value stored in that cubicle, so that return bookCase[cubicle#].

An object is similar to an array, as it contains a certain amount of properties and values, but you can target those properties by stating its key (in the form of a string) using either dot or bracket notation. For instance, let's take the object:

schoolLocker {
text book: Science,
lunchBox: empty
}

If you want to see what text book is in your locker, you'd write return schoolLocker["text book"];. Checking the status of you lunchbox would be return schoolLocker.lunchbox;.

The notation you use is important though. Whilst dot notation looks a lot cleaner and easier to read, it relies on you know the property beforehand. I you are accessing an objects data in a function by getting passed a property, you'd be best to use bracket notation, otherwise with dot notation you'll get an undefined error if that property doesn't exist.

JS functions: The automaton

Functions are great for automating repetitive tasks. They are reusable pieces of code, that whenever they are invoked (called), they perform a series of tasks dependent on their inputs and conditions. There is no limit to how many times a function can be called on a specific page.
